Diagnóstico do Keeper of Realms

Problema Identificado

Identificamos que você está tentando acessar páginas HTML estáticas diretamente, mas o projeto atual é uma aplicação React que utiliza roteamento do lado do cliente.

Atenção: Acessar diretamente arquivos como /home.html, /shop.html, etc. não carregará a aplicação React completa, resultando em telas em branco ou comportamento inesperado.

Explicação do Problema

O projeto Keeper of Realms Forge tem duas abordagens diferentes:

  1. Páginas HTML estáticas na pasta public/ - estas são páginas simples que não fazem parte da aplicação React principal.
  2. Aplicação React com sistema de rotas - esta é a aplicação principal que deve ser acessada pela URL base.

O servidor Vite está configurado para servir a aplicação React na raiz (http://localhost:3000/) e os arquivos estáticos diretamente de seus caminhos (http://localhost:3000/arquivo.html).

Solução

Para acessar a aplicação React completa: Use sempre a URL base http://localhost:3000/ e navegue usando os links internos da aplicação.

A aplicação React utiliza o React Router para gerenciar as rotas. Quando você acessa diretamente uma URL como http://localhost:3000/shop.html, você está tentando acessar um arquivo estático, não uma rota da aplicação React.

Rotas Disponíveis na Aplicação React

Rota Descrição
/ Página Inicial
/login Página de Login
/register Página de Registro
/shop Loja
/missions Missões
/inventory Inventário
/tables Mesas de Jogo
/teste Página de Teste (nova)

Importante: Não confunda as rotas da aplicação React (/shop) com os arquivos HTML estáticos (/shop.html).
